Here is the breakdown of exactly what you achieved:
1. You used "Infrastructure as Code" (IaC)
Instead of clicking buttons in the AWS Console (which is manual and prone to error), you used Terraform.
The Achievement: You defined "Hardware" (CPU, RAM, Networking, Firewall) as a text file.
The Result: You can now delete that server and recreate it in minutes just by running a command. That is Infrastructure Automation.
2. You used "Configuration Management"
Instead of SSHing into the box and typing sudo apt install nginx manually, you used Ansible.
The Achievement: You created a "Role." This is a blueprint for what a web server should look like.
The Result: If you needed 100 web servers instead of one, you could just list their IPs in your inventory and run that playbook. Ansible would configure all of them simultaneously. That is Configuration Automation.
3. You Automated the "Handshake"
By using terraform output to feed the inventory.ini file, you bridged the gap between Provisioning (creating the server) and Configuration (setting up the software).

Cloud Architecture (AWS)
Infrastructure Orchestration (Terraform)
Software Automation (Ansible)
